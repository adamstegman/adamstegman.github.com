Deployer

Automatic project deployment based on push notifications

idea

Started on July 29, 2011

Open source. Code available here.

Deployer is a Mac menubar application that automatically deploys a project when it receives a notification to do so.

Previously my blog was on Tumblr , so I could update it from anywhere using the iPhone application. Now, since I host my website and blog myself now and keep all the code on Github , I need to keep on top of updates.

To do that, I want to be able to write a post from anywhere. I found the right platform in Nanoc::Heroku which you can read more about on the AdamStegman.com project page. Nanoc::Heroku supports Markdown pages, which I could easily write from my iPhone or iPad.

Unfortunately, there are two key steps missing once I've written something on a mobile device.

  • Commit the post to Github
  • Run the deploy script

Currently, neither of those is possible from an iOS device (as far as I can tell). To remedy this, I'm writing an application for each. I'm starting with the less ambitious of the two, running the deploy script.

Using a service hook from Github, I hope to receive commit notifications and automatically update and deploy the website without any interaction on my part.

I don't know yet how I will receive the notifications. Boxcar has a stream API but I'm not sure yet if that will work.
